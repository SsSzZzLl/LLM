CDS547 Introduction to Large Language Models — Course Notes (Sample Corpus)

Course overview:
CDS547 surveys transformer architectures, pre-training objectives, prompting, fine-tuning, and retrieval-augmented generation. Students complete readings from NeurIPS and ACL papers and a group project demonstrating an LLM application or analysis.

Retrieval-augmented generation (RAG):
RAG conditions a generator on retrieved passages from a document collection. Lewis et al. (NeurIPS 2020) introduced RAG for knowledge-intensive tasks. A typical pipeline chunks documents, embeds chunks, retrieves top-k passages for a query, and prompts the language model to answer using only that context.

Evaluation themes:
Faithfulness measures whether an answer is supported by provided context. Hallucination refers to fluent but unsupported claims. Comparing RAG to a no-retrieval baseline helps quantify grounding improvements on factual QA.

Chunking practice:
Smaller chunks improve precision but may lose context; overlap between adjacent windows reduces boundary errors. Hybrid retrieval combining sparse (BM25) and dense (embedding) signals often improves robustness on short queries.

Responsible use:
API keys must be kept private. Teams should document model names, dates, and temperature settings for reproducibility.
